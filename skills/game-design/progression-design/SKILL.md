# 养成设计 Game Progression Design

## 何时使用
- 增加任何形式的解锁、等级、升级、精通（mastery）、装备词条层
- 玩家在两次会话（session）之间缺少「下次开局想干嘛」的目标
- 内容更新节奏需要更强的奖励结构来支撑留存
- 现有养成线过于复杂，新玩家被劝退或老玩家有「死车道」

## 核心方法论
**Progression Layers（养成层）**：短期（单局内）/ 中期（数日）/ 长期（数周以上）三层养成，每层服务不同目的。
**Reward Pacing（奖励节奏）**：奖励到来的时间间隔与情绪曲线匹配，避免「干等」或「连续高潮疲劳」。
**Gating（门槛）**：用什么机制决定玩家何时解锁下一层（等级 / 通关 / 资源 / 时间）。
**Catch-up（追赶机制）**：让新玩家或回归玩家不至于被永久甩开。

## 工作流程
1. 定义养成层级以及每层如何反哺核心循环（core loop）
2. 设定奖励的时机、节奏、门槛规则
3. 防止「前期复杂度过载」与「死车道」（dead progression lane）
4. 识别 reset（赛季重置）、catch-up（追赶）、平衡（balance）风险
5. 规划监控养成健康度的遥测（telemetry）

## 检查清单
- [ ] 短/中/长期三层养成都有明确目标
- [ ] 每层都能讲清「玩家做完得到什么具体奖励」
- [ ] 奖励曲线模拟过至少 3 种玩家路径（新手 / 中手 / 高手）
- [ ] 无「死车道」——所有养成线都有玩家继续投入的理由
- [ ] Catch-up 机制存在（哪怕只是「保底掉落」）
- [ ] 遥测埋点覆盖：开启率、停留时长、放弃节点

## 常见失败模式
- 加入不服务核心循环（core loop）的养成线
- 规则模糊，工程师无法实现可验证的解锁条件
- 没有遥测就调养成曲线
- 前期堆太多层，新玩家蒙圈
- 长期养成无尽头，老玩家失去目标感

## 协作注意事项
- 接到任务前，先用 4 把尺子过一遍：**WHY 是否清楚 / 路径是否最短 / 决策是否能回答为什么 / 这句话能改变 K师决策吗**
- 任何对外交付物署名统一用：**马启航Marvis**
- 多种养成模型并存时，先画一张「玩家时间投入 → 收益」曲线图给 K师拍板
- 不确定门槛松紧时，先列宽松 / 标准 / 严格三档供选
