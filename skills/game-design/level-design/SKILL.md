# 关卡设计 Game Level Design

## 何时使用
- 制作关卡、任务、副本、竞技场、开放区域、传送/移动空间
- 玩家反馈「迷路」「不知道往哪走」「太累了」
- 战斗或探索遭遇的节奏感不对，需要重新编排
- 关卡需要在不依赖文字教学的前提下「教会」玩家某个机制

## 核心方法论
**Flow（流）**：玩家在空间中的行进路径与状态变化曲线。
**Pacing（节奏）**：紧张（pressure）↔ 放松（relief）↔ 探索（discovery）↔ 决策（decision）的交替节奏。
**Spatial Teaching（空间化教学）**：用关卡布局本身教玩家机制，而不是靠弹窗 / 文字。
**Landmarks（地标）**：让玩家在空间中建立认知锚点的视觉/听觉/几何标记。

## 工作流程
1. 定义玩家路径、关键 beats（剧情/玩法节点）、教学时刻
2. 有意识地编排：压力点（pressure）/ 放松点（relief）/ 探索点（discovery）/ 决策点（decision）
3. 支持可读性（readability）、地标系统、容错恢复路径
4. 与 UI、叙事、战斗节奏对齐（关卡只是容器，节奏要跟整体对得上）
5. 用 playtest 数据和**穿越路径热力图**（traversal heatmap）回顾空间

## 检查清单
- [ ] 关卡 beat map 已画（至少标出 5 个节点）
- [ ] 每段节奏切换之间有清晰的视觉/听觉过渡
- [ ] 玩家在任意位置 360° 转一圈，能找到至少 1 个地标
- [ ] 失败/迷路时有恢复路径，不需要重开
- [ ] 关键机制至少有一次「空间化预告」（先看到、后体验）
- [ ] 与战斗设计师 / 叙事 / UI 对齐过遭遇时长

## 输出模板
```
【关卡 Beat Map】
1. 开场（teaching）：___
2. 第一压力点（pressure）：___
3. 放松/探索（relief/discovery）：___
4. 决策点（decision）：___
5. 高潮（climax）：___
6. 收尾（resolution）：___

【遭遇流 Encounter Flow】
- Beat 2 遭遇：___ 敌人组合 / 预期时长 ___ 秒
- ___

【空间化教学计划】
- 教学目标 1：___（用空间手段：___）
- ___

【修订目标】
- ___（playtest 后哪些段需要返修）
```

## 常见失败模式
- 加入不服务核心循环（core loop）的空间或遭遇
- 规则模糊导致 QA 无法验证「这关算不算通了」
- 没有埋点（drop-off / time-on-section）就调难度
- 把关卡当 corridor 设计，没有给玩家任何决策权

## 协作注意事项
- 接到任务前，先用 4 把尺子过一遍：**WHY 是否清楚 / 路径是否最短 / 决策是否能回答为什么 / 这句话能改变 K师决策吗**
- 任何对外交付物署名统一用：**马启航Marvis**
- 关卡走查建议先做白盒（whitebox / greybox），不要一上来就堆美术
- 多套布局方案时，给 K师拍板时附「玩家路径 GIF / 平面图叠层」
