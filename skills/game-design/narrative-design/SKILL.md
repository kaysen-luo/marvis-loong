# 叙事设计 Game Narrative Design

## 何时使用
- 需要为故事、对话、世界观（lore）、角色弧光建立结构
- 玩家反馈「故事跟玩法没关系」「为什么我要做这个任务」
- 内容量级上升，设定开始出现自相矛盾，需要 canon 管理
- 即将开始本地化（localization）或语音录制（VO），需要前置规划

## 核心方法论
**Narrative-Gameplay Alignment（叙事-玩法对齐）**：故事节拍必须与玩法节拍咬合，叙事不是包装纸，而是动机引擎。
**World Rules（世界规则）**：世界观需要 3-5 条不可违反的硬规则，所有故事都从这里推导。
**Dialogue Intent（对话意图）**：每段对话先回答「玩家听完该有什么情绪 / 动作」，再写文本。
**Canon Reference（设定圣经）**：人名、地名、术语、时间线一处统一维护，避免「同一个 NPC 三种名字」。
**Localization Readiness（本地化预备）**：写作阶段就考虑变量、性别、复数、文化禁区。

## 工作流程
1. 定义叙事目标、主线弧光、世界规则
2. 把故事节拍与玩法目标流（objective flow）逐一对应
3. 写作或大纲对话时，预先考虑实现约束（字数、变量、UI 容器）
4. 建立 canon 参考文档与命名一致性规则
5. 早期就规划字幕、VO、本地化影响（成本与排期）

## 检查清单
- [ ] 世界硬规则 ≤ 5 条，每条都能反例
- [ ] 每个主线节拍都能对应一个玩家可执行动作
- [ ] 对话稿包含：意图 / 文本 / 触发条件 / 后果
- [ ] Canon 文档建立，至少覆盖：主要角色、地点、术语、时间线
- [ ] 本地化约束清单（变量、性别、复数、禁区）已交付翻译方
- [ ] VO 排期对齐，识别需要预录与可后录的段落

## 输出模板
```
【叙事 Brief】
主题：___
世界硬规则：
  1. ___
  2. ___
  ...

【故事-玩法咬合表】
| 节拍 | 玩家动作 | 情绪目标 | 对应玩法系统 |
| --- | --- | --- | --- |

【Canon 注记】
- 角色 ___：___
- 地点 ___：___
- 术语 ___：___

【本地化 / VO 注意事项】
- ___
```

## 常见失败模式
- 增加不服务核心循环（core loop）的剧情系统（看完跳过的过场）
- 规则模糊，玩家不知道「这个世界到底允不允许 X」
- 没有 canon 文档，新成员上来就破坏设定
- 文本写完才想起本地化，翻译方掉链子
- 对话写得很美但玩家不知道听完该干嘛

## 协作注意事项
- 接到任务前，先用 4 把尺子过一遍：**WHY 是否清楚 / 路径是否最短 / 决策是否能回答为什么 / 这句话能改变 K师决策吗**
- 任何对外交付物署名统一用：**马启航Marvis**
- 多套叙事方向给 K师拍板时，每个方向附「Logline（一句话剧情核）」+「玩家情绪曲线」
- 涉及文化敏感话题主动标红，请 K师确认
