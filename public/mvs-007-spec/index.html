CHAPTER · 01

一句话规格 & 唯一验证问题

一个玩家，操控步兵 Trooper，在 10 分钟内打完 5 波异形 + 1 个 Boss，期间通过 6 次三选一让 Build 一直在长——验证「爽感是否成立」。

MVP 唯一要验证的问题

?? THE QUESTION 「一个人打 10 分钟外星怪，Build 一直在长，最后打 Boss——到底爽不爽？」

爽 → 加人、加卡、加关、加化学反应、加装备、加探索，滚向 V1.0。
不爽 → 回炉重造核心循环，不做后面任何东西。

这是唯一的成败标尺。MVP 期间所有决策以「是否服务于验证这个问题」为准。

为什么砍掉 X（V1.0 完整版有但 MVP 无）

V1.0 有MVP 砍砍掉的理由
4 个英雄1 个 Trooper1 英雄足以验证核心爽感；多英雄价值只在化学反应存在时显现
5 个关卡1 关 10 分钟1 关够验证节奏 / 弧线；多关是「内容扩展」不改核心循环
40+ 张升级卡10 张10 张 = 6 次三选一 = 每次 3 个像样选项，已足够验证 Build 感
化学反应无1 英雄 + 10 卡的 Build 空间还谈不上「协同网」
装备系统 / 地图探索 / 组队无都是二级系统，MVP 装不下

砍掉的一切共同理由：MVP 只回答一个问题。任何不服务于这个验证的东西，都是干扰项——无论它在 V1.0 里多重要。

CHAPTER · 03

核心玩法循环（Core Loop）

半自动割草：玩家控移动 + 主动技能，主武器自动开火。10 分钟一局，节奏靠 wave 解锁 + 升级驱动三选一。

4 个 Verbs（核心动词）

#Verb输入方式手动 / 自动教学时机
1移动左半屏触摸摇杆 / WASD手动开局 3 秒学会
2主武器（等离子步枪）无需操作 · 射程内自动自动落地即懂
3能量脉冲右下按钮 · r=105 px 命中区手动05:00 Spitter 出场后学「群清 + 打断」
4战术翻滚 v0.2 实装右侧按钮手动03:00 Rusher 出场后学「无敌帧躲冲刺」

设计意图：4 个动词是新手 10 分钟内能全部掌握的极限。少一个 = 无聊，多一个 = 认知过载。
v0.1 白模注意：战术翻滚 spec 有但 code 未实装（当前唯一无敌是碰怪后的 300ms i-frame，被动触发）。v0.2 补齐。

10 分钟单局节奏弧线

抽卡节奏（升级驱动，非固定 90 秒）

白模实现：每次角色升级 → 暂停世界 → 三选一 → 应用 → 恢复（openLevelUp()）。
升级门槛 xpToNext = 20 + (level-1) × 10——Lv2 需 20 xp，Lv3 需 30 xp……Lv10 需 100 xp（累计 540 xp）。
预期一局 6-8 次抽卡，与 MVP 规格卡「6 次三选一」意图对齐。

★ 射程限制 · 500 px（K师 2026-07-15 拍板）

建议值 · 500 px（约屏幕高 1/3 · 1920 竖屏下 = 500/1920 ≈ 26% 屏高）

规则：超射程外的敌人不吃 nearestEnemy()——步枪不再自动开火，玩家必须走近才输出。

为什么要加：

  • 白模现状是无限射程自动锁最近敌人——玩家可以龟缩不走位，主武器一直响，「移动」这个 Verb 被架空。
  • 加射程后强迫「移动 + 输出」双 Verb 咬合：远处刷出来的怪 你不走过去它就打不到。走位不再只是躲，也是「造成输出」。
  • Spitter 的 900 px 攻击射程反而 比你远——玩家必须主动接近才能干掉炮手，制造 「顶着酸弹前压 vs 龟缩挨骚扰」 的抉择。
  • 半自动割草的爽感来自「走位」和「输出」同时发生，不是「站桩自动泼」。

数值理由：屏幕高 1920，玩家在中心时上下各 960 px。500 px ≈ 覆盖玩家周围 1000×1000 的战斗圈，够看清目标、够输出，但看不到屏幕最边缘。留出「战术真空」——那些还没进入射程但正在压过来的怪，是压力来源。

实装点：nearestEnemy() 内加一行 if (bestD > 500*500) return null；同步 UI 显示半透明 500 px 圆圈（淡青蓝，只在 射程内无敌人 时高亮提示「你需要移动」）。

各敌人设计角色

Crawler · 数量海教学怪

开局预放 5 只。每 1200 ms 刷一波，波数随时间递增——wave = min(4, 1 + floor(elapsed/120000))：0-2min 一波 1 只，2-4min 一波 2 只，4-6min 一波 3 只，6+min 一波 4 只。07:30 起触发 crawlerDense，波数 ×2（可能一波 8 只）。
它的角色：教玩家「主武器 = 一切」。爬虫慢、伤害低，只是让玩家感受到击杀反馈。

Rusher · 冲刺威胁 · 教走位

03:00 解锁（elapsed >= 180000）。每 5 000 ms 刷 1-2 只。09:00 起触发 rusherDense，数量 ×2。
它的角色：教玩家躲。速度 220 px/s（比玩家的 240 稍慢但接近）+ 撞后自爆 = 玩家如果不移动就会挨 15 伤。这是翻滚技能（v0.2）的教学怪——当前玩家只能靠 i-frame 硬吃或纯走位躲。

Spitter · 远程压制 · 教群清

05:00 解锁（elapsed >= 300000）。每 7 000 ms 刷 1 只。每 1500 ms 全场 Spitter 集体对 <900 px 玩家吐一发酸弹。
它的角色：教玩家用脉冲。它慢（60 px/s），HP 高（50，5 倍于爬虫），但射程 900 px 比玩家的 500 px 主武器远——玩家龟缩就打不完酸弹。必须前压近战或用脉冲一次群清一片。

抽卡权重 / 刷新规则

白模 pick3(state) 实现：

  1. 从卡池过滤掉已选过的 once 卡（pool.filter(c => !(c.once && s.cards.includes(c.id)))）
  2. Fisher-Yates 洗牌
  3. 取前 3 张
  4. 若池子剩 <3 张，少给几张

MVP 权重均等——每张卡出现概率相同。V1.0 可加稀有度分层（白 / 蓝 / 紫 / 橙）。

卡池设计原则

「6 次三选一，每次都在选」
  • 卡强度接近，没有明显最优解——玩家真的在选，而不是「必拿 X 卡」
  • 类型分布 4 / 3 / 3（主武器 / 副技能 / 光环）——每次抽卡大概率覆盖 2-3 个类型
  • 4 张可叠加 + 5 张 once——保证后期还有可抽的，同时防「一辈子叠射速」
  • 吸血 / 装甲 / 移速三张生存卡—— Boss 前的救命关键（当前无 Boss，仍是 08:00 后压力峰值救命卡）

出场时机 08:00 · 目标击破 90-120 s · 3 阶段综合考核——考察走位、脉冲群清、生存 Build 是否够硬。

3 阶段状态机

阶段 1 · 冲撞（HP 100%-66% · 前 30-40 秒）

  • 行为：直线冲撞玩家（3 秒蓄力预警 → 4 秒 400 px/s 冲刺 → 3 秒硬直）
  • 撞伤 60（60% 玩家血量，一次挨则濒死）
  • 教学：走位 / 翻滚（v0.2）躲冲撞路径

阶段 2 · 召唤（HP 66%-33% · 中段 30-40 秒）

  • 行为：每 8 秒吼一次，从体内孵化 Crawler ×6（半径 60 px 圆内散开）
  • Boss 自身减速到 20 px/s，重点是清杂
  • 教学：能量脉冲的战略价值——一发脉冲能一次干掉半个孵化圈

阶段 3 · 酸雨 AoE（HP 33%-0% · 后 30-40 秒）

  • 行为：每 4 秒随机地面点位落下 3 个酸雨圆（半径 120 px · 2 秒预警 → 引爆 · 40 伤 · 持续酸池 3 秒）
  • Boss 停止移动，站桩——玩家必须一边输出一边不停走位
  • 教学：Build 是否够硬直接决定这段能不能扛过——生存卡的价值验证

Boss 是 MVP 唯一的「反 Roguelite 自动泼」检查点——玩家不能只靠自动开火过关，必须手动决策 3 件事：
