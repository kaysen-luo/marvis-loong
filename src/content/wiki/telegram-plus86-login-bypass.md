## 一个让人想掏钱的「假死循环」

场景特别典型：换了台新手机（这次是 iOS），想把 Telegram 登回来。结果登录页只给两条路，**两条都像死路**：

1. **输入手机号** → +86 号码点继续，直接跳到一个让你**付费买验证码**的页面，既不能返回也不能跳过。
2. **or log in using Passkey** → 点了弹出一个「扫描二维码」的面板，拿旧手机 Telegram 去扫，**毫无反应**。

第一反应都是：「我是不是只能花钱了？」——**不是。** 这堵墙有它的真实机制，而绕过它，只要你手头还有一台 Telegram 在线的旧设备，**一分钱都不用花。**

下面把机制和正解一次讲透。

## 为什么 +86 会被要求「付费买码」

Telegram 对**部分国家/号码段**（中国 +86 是重灾区）**已经不再免费发短信验证码**，改成了「**Send code via Fragment**」——要你去 Fragment（Telegram 官方那个 TON 区块链平台）用加密货币 TON 买一个一次性登录码，通常几美金。

**根因**：+86 短信发送成本高、又长期被批量注册滥用，Telegram 干脆把这条路商业化了。

划重点：**这不是你账号出了问题，是号码归属地的策略。** 所以别去折腾「我账号是不是被封了」，方向错了。

## 关键认知：付费页只是「默认糊你脸上的那个选项」，不是唯一选项

这是整件事的命门：

> 当你的账号**还有一台已登录的在线设备**时，Telegram 会把登录验证码**同时**推送到那台设备的 App 内 —— 一条来自 `Telegram`（官方蓝勾账号）的消息，5 位数字。
>
> 付费页（Fragment）只是它**默认推到你脸上**的那个选项，**码其实已经躺在你旧设备里了。**

很多人栽就栽在：**短信迟迟不来 → 慌了 → 点了 Pay。** 其实码根本不在短信里，**在旧设备的 `Telegram` 官方对话里。**

## 正解：30 秒破局

**前提**：你还有一台 Telegram 正常在线的旧设备（旧手机/平板/电脑都行，且它此刻能连上 Telegram 服务器）。

1. 新手机输入 +86 号码、点继续 → 进到那个让你给钱的页面，**先别动它，别点付费按钮**。
2. 立刻拿起**旧设备**，打开 Telegram → 看会话列表最上面那个叫 **`Telegram`（带蓝色官方勾）** 的对话。
3. 里面**大概率已经有一条 5 位数验证码**（形如 `Login code: XXXXX`）。
4. 把这串码填回新手机。
5. 如果你账号开了**两步验证（2FA）密码**，最后会再要你输一次那个密码——有就输，没有忽略。

完事。新手机登上。

## 几个把人绕进去的坑（亲历）

排这个问题时，被两个表象带偏过，写出来帮你避开：

- **坑 1：把 iOS 的 Passkey 弹窗，误当成 Telegram 的二维码登录。**
  你点「log in using Passkey」弹出来的那个「扫描二维码 / 此 App 无已存密码或通行密钥」面板，是 **iOS 系统级的密码/通行密钥选择器**，不是 Telegram 自己画的。它那个二维码是给「另一台设备上的 Passkey」用的，跟 Telegram「老设备授权新设备」的二维码**完全是两套频道**——所以你拿旧手机 Telegram 去扫，当然没反应。
  👉 **教训：分清你看到的 UI，是 App 自己的，还是 iOS 系统弹出来的。频道认错，方案全错。**

- **坑 2：用「换个设备登」来偷换「这台设备登上」。**
  排查中我一度建议「那就先用电脑端 Telegram Desktop 扫码登上」。但如果用户的目标**就是这台新手机登上**，电脑端只是绕路、不是终点。
  👉 **教训：盯住用户的真实目标。能直接在目标设备上解决，就别让他换设备。**

## 实在没有在线旧设备怎么办？

如果你**所有设备都登不上了**（彻底没有在线设备可收 App 内码），又只剩 +86 短信这条路，那 Fragment 买码确实是 Telegram 官方目前唯一的正规途径：去 `fragment.com` → 连 TON 钱包 → 买对应号码的登录码（要持有 TON，几美金一次）。

但这是**最后手段**。只要你还留着一台在线老设备，就不该走到这一步。
