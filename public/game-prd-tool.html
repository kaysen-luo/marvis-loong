12 模块

    01项目元信息

    最基本的身份卡。所有 AI 子代理读 PRD 第一眼看这里。

    02核心玩法

    核心循环 = 5 秒能讲完的事。例: “射击-躲门-升级武器-打 BOSS”。

    动词越少越好, 动词应该是玩家背背背重复做的。

    03平台与输入

    选定形态后其他一切 (UI/手感/布局) 都要跟这里对齐。

    04数值表

    所有可调参数值都放这里, 支持动态加行。AI 看表比看话清楚。

    提示: 点单元格可编辑 · 点右侧 × 删行

    05视觉规范

    点颜色块会弹出原生 color picker 。颜色名主要让 AI 读 (例: “主色/进度色/警告色”)。

    06关卡/场景

    按时间轴 (00:00 / 00:10 / 00:30...) 或按 stage 列。表达节奏、老拍。

    07系统/机制

    每个机制一张卡片: 名字 / 输入条件 / 输出效果 / 视觉表现 / 数据 schema / 不允许做的。

    08UI/HUD 元素

    顶bar / 底bar / 浮层 / 数字位 — 每项标清位置 + 内容 + 字号。

    09资源清单

    图/音/字体 — 一项一行, 支持加行。对付 subagent 要主动取素材的场景。

    10验收/自检

    验收 = “我怎么知道这个做对了” 。自检项 = 可勾选的检查项。

    11内容铁律

    “不允许做的” — 防 subagent 自作主张。这个区块会以 > ⚠️ blockquote 输出到 markdown 。

    12参考材料

    竞品 + 风格 reference + K师手绘草图。手绘可上传, 存为 base64 进入 JSON
