# 经济平衡 Game Economy Balancing

## 何时使用
- 游戏里有任何形式的货币、资源、合成、商店、升级经济
- 奖励发得太多（玩家失去目标感）或太少（玩家挫败）
- 已上线游戏需要做 live tuning，但缺一套结构化框架
- 新增一个货币 / 兑换路径前，需要评估对存量经济的冲击

## 核心方法论
**Currency Roles（货币角色）**：每种货币只能服务一个清晰目的（软通货 / 硬通货 / 时限资源 / 进度积分等）。
**Sources & Sinks（来源与消耗点）**：每种货币的进入路径和退出路径必须成对设计，缺哪边都会出问题。
**Inflation Control（通胀控制）**：通过 sink、时限失效、上限封顶等手段防止资源贬值。
**Tuning Plan（调参计划）**：哪些数值可以线上拨动，每个的影响半径多大，必须事先想清楚。

## 工作流程
1. 定义每种货币的角色和允许的流向（roles & flows）
2. 画出 sources / sinks / 兑换路径矩阵
3. 设定基线数值（baseline values）和反通胀机制
4. 识别滥用案例（abuse cases）、卡死状态（softlocks）、过度奖励循环
5. 为经济关键事件埋点（telemetry），监控线上健康度

## 检查清单
- [ ] 每种货币能说清「它存在的唯一理由」
- [ ] Source / Sink 矩阵成对（无单边货币）
- [ ] 至少 2 种通胀控制机制已设计
- [ ] 至少识别 3 种滥用案例和应对方案
- [ ] 经济关键事件（获取、消耗、兑换）已埋点
- [ ] 调参计划列明每个变量的「可拨动范围」和「拨动后预期影响」

## 常见失败模式
- 增加了不服务核心循环（core loop）的货币 / 经济系统
- 规则模糊导致工程实现凭感觉
- 没有埋点就开始调数值
- 货币种类过多，玩家无法理解每种的用途
- Source 设计完忘了 Sink，造成长期通胀

## 协作注意事项
- 接到任务前，先用 4 把尺子过一遍：**WHY 是否清楚 / 路径是否最短 / 决策是否能回答为什么 / 这句话能改变 K师决策吗**
- 任何对外交付物署名统一用：**马启航Marvis**
- 经济调整建议附「线上回滚方案」，所有 live 改动必须可回退
- 给 K师拍板时附「调参前/后玩家路径模拟」对比表，不要光给数值
